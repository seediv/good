| ID | 原則名 | 元ID | 説明 |

| TI-06 | 圧縮による余白 | A2 | 何を削れば本質が際立つか |
| TI-07 | 意図の先行固定 | A3 | 成功とは何か先に定義できるか |
| TI-08 | 階層的抽象化 | B1 | 何を隠し、何を見せるか |
| TI-09 | 次元の変換 | B2 | 別の表現で保存できるか |
| TI-10 | 小さな反復 | I1 | 最小の動く単位は何か |

```yaml
translation_integrity:
  name: 翻訳完全性パターン
  description: 意図を実装に翻訳する際の品質を保証する
  elements:
    - TI-01  # 制約による解放
```

## 3つのメタ原則について

TIPには3つのメタ原則（M1-M3）があるが、これらはLayer 0への追加ではなく、**参照情報として保持**する：

| メタ原則 | 内容 | Layer 0との関係 |
|---------|------|----------------|
| M1 意図の優位性 | ルールより目標を優先 | MP-01「含んで超える」と親和 |
| M2 次元的効率性 | 本質を失わずに圧縮・変換 | MP-04「視覚経済」と親和 |
| M3 関係性エンコーディング | 構造はコミュニケーションに従う | 新規観点 |

**理由**: Layer 0は安定した根本原理。TIPのメタ原則は検証済みだが、Layer 0に追加するには更なる熟成が必要。
