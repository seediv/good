# TIP クイックスタート
# 5分で試す翻訳完全性原則

## 今すぐ試す：核心4原則

### お題を決める

何でもいい。例：「傘」「会議」「コーヒー」「転職」

---

### C1: 制約による解放

**問い**: これは何の構造を持っているか？

```
傘の場合：
✗ 雨を防ぐ（機能）→ 浅い
○ 境界を作る（構造）→ 深い

「傘は境界を作る構造である」
```

**コツ**: 動詞（〜する）より名詞（〜である）を探す

---

### C2: 結合規約の単純さ

**問い**: その構造は何と何を繋ぐか？

```
傘の場合：
「境界」は何と何を分けるか？
→ 内側（私）と外側（世界）を繋ぐ

「傘は私と世界の境界である」
```

---

### C3: 構造に検証を埋め込む

**問い**: 「〜でなければ〜と分かる」の形にできるか？

```
傘の場合：
「境界がなければ、どうなるか？」
→ 私と世界が区別できなくなる

「境界がなければ、自分がどこまでか分からない」
```

---

### C4: 創発の余地を残す

**問い**: この構造を別の文脈に適用すると？

```
傘の場合：
「境界」を人間関係に適用すると？
→ 人は皆、見えない傘を持っている

「人は皆、見えない傘を持っている。
 近づきすぎると濡れる。離れすぎると届かない。
 良い関係とは、傘の端が触れ合う距離である。」
```

---

## 出力例

### 入力: 傘

### 出力:
> 「人は皆、見えない傘を持っている。近づきすぎると濡れる。離れすぎると届かない。良い関係とは、傘の端が触れ合う距離である。」

---

## よくある失敗

| 失敗 | 原因 | 対策 |
|------|------|------|
| 出力が浅い | C1で機能を選んだ | 構造を選び直す |
| 意味が繋がらない | C2をスキップした | 関係性を明示する |
| 当たり前で終わる | C3をスキップした | 検証の視点を入れる |
| 既存の比喩になる | C4で借用した | 自分で構造から導く |

---

## フェーズ別の使い方

| 状況 | 使う原則 |
|------|---------|
| 企画書を書く前 | A1本質保存, A3先行固定 |
| 設計する時 | C1制約, C2規約, B1階層 |
| 実装する時 | C3検証, I1小反復 |
| 改善する時 | E1透明可逆, E2段階習熟 |
| 常に意識 | X1文脈一回性, X2関係性 |

## 5分チャレンジ

以下のどれかで試してみる：

1. **どらやき** → 人間関係に適用
2. **時計** → 約束に適用
3. **階段** → キャリアに適用
4. **窓** → コミュニケーションに適用
